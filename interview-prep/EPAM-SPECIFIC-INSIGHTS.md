### Round 2: Technical Deep Dive (1.5-2 hours)

**This is the MOST IMPORTANT round**

**What to Expect:**
The interviewer (usually a senior architect/lead) will:

1. Start with basic concepts and go DEEPER
2. Test your fundamentals thoroughly
3. Ask you to code 2-3 problems
4. Evaluate your problem-solving approach
5. Check communication skills continuously

**Real Questions from EPAM Interviews:**

**JavaScript/TypeScript:**

- Explain closures with a real-world example
- Difference between `var`, `let`, and `const` (with hoisting)
- Event loop explanation (be very clear on this)
- Promises vs async/await (when to use which)
- `this` keyword in different contexts
- Prototypal inheritance
- How does garbage collection work?

**Data Structures:**

- Implement stack using array
- Reverse a string using recursion and stack
- Find first common node in two linked lists with common tail
- Implement all three cases (equal length, first longer, second longer)
- Binary search implementation
- Quick sort implementation with analysis

**OOP & Design:**

- SOLID principles with examples
- Design patterns you've used (real projects)
- Difference between abstraction and encapsulation
- When to use composition vs inheritance
- How would you design [specific system]?

## ðŸš« Common Reasons for Rejection

Based on real candidate feedback:

1. **Poor Communication**
   - Coding in silence
   - Not explaining thought process
   - Can't articulate concepts clearly in English

2. **Weak Fundamentals**
   - Memorized solutions without understanding
   - Can't explain time/space complexity
   - Doesn't understand OOP principles deeply

3. **Giving Up Too Easily**
   - Not trying different approaches
   - Ignoring hints from interviewer
   - Saying "I don't know" without attempting

4. **Code Quality Issues**
   - Messy, unreadable code
   - No error handling
   - Doesn't test code
   - Poor variable naming

5. **Unprofessional Behavior**
   - Late to interview
   - Not prepared
   - Negative talk about previous employer
   - Overconfidence or arrogance

## âš¡ Last-Minute Preparation Tips

### 1 Day Before Interview:

**Morning:**

- Review OOP principles (inheritance, polymorphism, encapsulation)
- Practice explaining event loop
- Review SOLID principles with examples

**Afternoon:**

- Solve 2-3 medium problems on arrays/strings
- Practice implementing stack, queue
- Review your resume projects thoroughly

**Evening:**

- Prepare 5 questions to ask interviewer
- Set up interview environment (quiet room, good internet)
- Test camera/microphone
- **Sleep well** - most important!

### Interview Day:

**30 mins before:**

- Warm up voice (practice speaking English out loud)
- Review your "Tell me about yourself" answer
- Do 1 easy coding problem to get in the zone
- Have water ready
- Bathroom break

**During Interview:**

- Greet warmly and smile
- Take notes during problem description
- Ask clarifying questions
- Think out loud
- Stay calm even if stuck
- Thank interviewer at the end

of the business value and user impact before diving into implementation.
This helps me prioritize better and deliver more business value."
